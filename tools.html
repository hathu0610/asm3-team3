Comment on audit trails

Looking at the audit trails, we can see that our team members have commit codes on time with clear commit messages. The audit trails also partially reflected our group’s work, due to the fact that most of the time we worked on Google Docs and discussed our problems online. The two most prominent problems that were reflected on the trails are the subpage set up and the animation. Instead of putting all the content in one page like assignment 2, for this assignment we decided to split our content into multiple subpages. This resulted in a conflict when setting up the navigation between the webpage. Normally in web development, if you want to add a link that navigates the user from the subpage back to the main page, you would put the href of the main page as “/”, since this is the main page’s url. However, in github pages, the main page’s url is “[username].github.io/[projectname]”, and when you add the href as “/”, this will navigate to the url “[username].github.io/”. This took a long time for us to resolve, due to the fact that most of the answers online suggested that we used the href as “/[projectname]” or “/index.html”. In the end, it turned out that we have to use the href as “./”. This can be clearly shown in the audit trails, as we had to try a few different ways before finding out the right answers. Another problem we encountered that was reflected in the audit trails was with the animation. For assignment 3, we also wanted to learn more about CSS, as thus we decided to add more animation to the content of the website. Setting up the animation was very difficult, since we have to deal with the CSS keyframe and position. For example, in order to have an element fading out in 5 seconds, one would have to set a keyframe to change the element’s opacity and add the 5-second timer in webkit-animation. We encountered some conflict with the webkit-animation, and fortunately in the end we found an easy-to-use CSS animation library called AOS - animate on scroll. This situation was not clearly shown in the audit trails because we add the animation while also updating the content for the timeframe. Overall, the audit trails reflect that we were able to gain new knowledge about web programming from assignment 2 - including creating subpages and adding animation.
